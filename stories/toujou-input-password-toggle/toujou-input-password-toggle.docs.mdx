# Toujou Input Password Toggle

A wrapper component that adds a show/hide toggle to a password input. It manages the input type switching between `password` and `text` and shows the appropriate toggle button based on the current visibility state.

## Slots

| Slot | Description |
|---|---|
| `input` | The password input element. Must be an `<input>` element |
| `show-password-button` | Button shown when the password is hidden. Clicking it reveals the password |
| `hide-password-button` | Button shown when the password is visible. Clicking it hides the password |

## Behavior

- On load the input type is `password` and the `show-password-button` slot is visible
- Clicking the show button sets the input type to `text` and switches to the `hide-password-button` slot
- Clicking the hide button sets the input type back to `password` and switches back to the `show-password-button` slot
- Click events on the toggle buttons are prevented from bubbling to avoid unintended form submissions

## Accessibility

- Use `aria-label` on both toggle buttons to provide meaningful descriptions for screen readers
- The `type="button"` attribute on the toggle buttons is important to prevent accidental form submission
- Consider adding `autocomplete="current-password"` to the input for better browser and password manager support
